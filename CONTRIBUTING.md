# How feedback is handled

1. Feature requests live in **Discussions → 💡 Feature Requests**; the community upvotes them.
2. When an idea gains traction, a maintainer opens a **Linear** issue and links it back to the discussion.
3. The maintainer replies on the discussion and applies a status label: `planned`, `in-progress`, or `shipped`.
4. On release, the pinned **“How this works + Roadmap”** post (Announcements) is updated.

Please keep **one idea per discussion** so voting stays meaningful. Thanks for helping shape mxalias!
